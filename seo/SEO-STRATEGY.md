## 1. Executive Summary

Flex Commerce occupies a strong regional position as a Shopify Plus partner agency in Manchester and Liverpool. The UK Shopify agency market is competitive but fragmented — no single agency dominates all keyword clusters. The highest-leverage SEO opportunity is a three-pillar attack:

1. **Service pages** — rank for high-commercial-intent keywords where decision-makers search (e.g., "Shopify Plus agency UK", "Magento to Shopify migration")
2. **Location pages** — capture city-level searches as the agency expands (10 UK cities planned)
3. **Content authority** — build topical depth through guides, checklists, and thought-leadership articles that funnel to service pages

The site already has a solid technical foundation (Next.js 14 static export, Cloudflare Pages, structured data, canonical tags). The primary gaps are E-E-A-T signals, case study depth, internal linking coherence, and content coverage in mid-funnel informational clusters.

|---|---|---|

| Commercial intent | 25% | Direct service inquiry = 10; awareness = 3 |

## 6. E-E-A-T Strategy

Google's Experience, Expertise, Authoritativeness, and Trustworthiness signals are critical for a professional services site.

### 6.1 Experience Signals (currently weak)

- **Action:** Publish 8+ detailed case studies with real client metrics (conversion rate lifts, revenue increases, speed improvements). Use client name where permitted; industry + city where not.
- **Action:** Add a "results" counter section to the homepage — e.g., "£50M+ revenue generated", "200+ stores launched", "50+ 5-star reviews".
- **Action:** Add project portfolio images with alt text to the `/work` section.

### 6.2 Expertise Signals (moderate; needs building)

- **Action:** Create individual author profiles for team members at `/about/team/[name]`. Each profile must include: headshot, bio, years of experience, Shopify certifications, LinkedIn URL.
- **Action:** Add `Person` + `ProfilePage` schema to each team member page with `sameAs` links to LinkedIn, Twitter/X.
- **Action:** All articles and guides should have a named author with a link to their profile.
- **Action:** Publish original research — e.g., "UK Ecommerce Migration Report 2026" — to establish citable data.

### 6.3 Authoritativeness Signals (weak; top priority)

- **Action:** Target 10 high-quality backlinks from UK ecommerce publications (Internet Retailing, Drapers, The Drum) in Year 1.
- **Action:** Guest post on Shopify's own blog and partner blogs (Klaviyo, Yotpo, Gorgias).
- **Action:** Submit to agency directories: Clutch, Trustpilot, Google Business Profile (both offices), Bing Places, Yell.com.
- **Action:** Speak at or sponsor Northern ecommerce events (MCR Ecommerce, NW Agency Awards) for brand mentions.
- **Action:** Issue press releases for major client wins (anonymised if needed) via PR Newswire UK.

### 6.4 Trustworthiness Signals (partially in place)

- **Action:** Ensure Trustpilot widget appears on all service pages, location pages, and the homepage.
- **Action:** Add `aggregateRating` schema to all service pages once review count reaches 10+.
- **Action:** Add `Review` schema items (3–5 per service page) with real client names and dates.
- **Action:** Display Shopify Partner badge prominently in header and on all service pages.
- **Action:** Add a clear Privacy Policy, Terms of Service, and cookie consent that meets UK ICO requirements.
- **Action:** Display physical address and phone number in the site footer (currently done in header/contact — confirm footer too).

## 7. Technical SEO Foundation

### 7.1 Strengths (current)

- Static export to Cloudflare Pages = excellent TTFB and global CDN delivery
- Next.js 14 App Router with per-page metadata API
- Canonical tags on location pages
- JSON-LD structured data: Organization, LocalBusiness, WebSite, Service, BreadcrumbList, FAQPage, Review
- `robots: { index: true, follow: true }` with Googlebot directives
- OpenGraph and Twitter card tags on all pages
- Inter + Playfair Display served via Google Fonts with `display: swap`

### 7.2 Gaps to Address

| Issue | Priority | Action |
|---|---|---|
| No XML sitemap confirmed | P1 | Verify next-sitemap or static sitemap.xml is generated at build time and submitted to Search Console |
| No robots.txt confirmed | P1 | Confirm robots.txt exists at root; ensure /out/ paths resolve correctly on Cloudflare |
| Breadcrumb schema missing from service pages | P1 | Add BreadcrumbList JSON-LD to all service, migration, and article pages |
| No `sameAs` on root Organization schema | P2 | Add social profile URLs to root schema (LinkedIn, Twitter, Instagram) |
| No `description` on root Organization schema | P2 | Add company description to root Organization JSON-LD |
| No `email` on root Organization schema | P2 | Add hello@flexcommerce.co.uk to root schema |
| Missing `aggregateRating` on service pages | P2 | Add once review count is sufficient |
| Article pages missing Article/BlogPosting schema | P2 | Add per-article schema with `author`, `datePublished`, `dateModified`, `headline` |
| Guide pages missing HowTo schema | P3 | Add HowTo schema to step-based guide pages |
| Checklist pages missing ItemList schema | P3 | Add ItemList schema to checklist pages |
| No FAQ schema on service pages | P3 | Add FAQPage schema to service pages where FAQ section exists |
| `lang="en"` should be `lang="en-GB"` | P2 | Update root layout HTML lang attribute |
| No hreflang (if targeting .co.uk specifically) | P3 | Add hreflang `en-GB` annotation if en-US content ever added |
| Image alt text audit needed | P2 | Ensure all `<Image>` components have descriptive alt text including target keywords |

### 7.3 Core Web Vitals Monitoring

The static Cloudflare Pages setup should comfortably pass Core Web Vitals. Monitor monthly via:
- Google Search Console (CWV report)
- PageSpeed Insights per key landing page
- Lighthouse CI in the build pipeline

Target benchmarks per service/location page:
- LCP < 2.5s
- CLS < 0.1
- INP < 200ms

## 9. Generative Engine Optimisation (GEO)

AI-powered search tools (ChatGPT, Perplexity, Google AI Overviews) increasingly answer agency comparison queries. To appear in AI-generated responses:

- Publish detailed, factual case studies with specific, citable metrics (e.g., "52% conversion rate increase in 3 months")
- Maintain consistent entity data: same name, address, phone across all directories and schema
- Use `Person` schema with `sameAs` LinkedIn for all team members
- Structure service descriptions with clear, quotable value statements
- Produce original research (e.g., UK Shopify Migration Report) that AI systems can reference
- Maintain accurate listings on Clutch, G2, Trustpilot (AI systems heavily cite these)
- Monitor AI citation monthly: search "Shopify Plus agency UK" in Perplexity and ChatGPT
